nss

A CLI utility for easily creating a (mostly) static node.js server on the fly

    Installation

    To install nss, first you must download the source code (or clone), and then run make install.

    $ git clone git://github.com/kbjr/nss.git
$ cd nss
$ [sudo] make install

    After installing, you will have a command nss that you can use to create your servers.

    Usage

    First, you create a new server which will have a document root of the current working directory.

    $ nss init

    You can then start the server and stop it again when you're done with it.

    $ nss start
$ nss stop

    Once the server is running, you can request your files at http://127.0.0.1:1337/. You can change the port number if you want, as well.

    $ nss config port 1234
$ nss restart

    Some more advanced features

    So, for the most part, this is just a static file server, but it has a few extra tricks in case you want a little more. For example, you can create a file with the extension .ejs and that file will be parsed with EJS templating. You can also do a little bit of more dynamic handling if you really want to using the controller editor.

    $ nss edit

    This will open up an editor with a very simple file that looks like this:

    module.exports = {

//  test: function(req, res) {
//      res.writeHead(200, {'content-type': 'text/plain'});
//      res.write('Hello, World!');
//      res.end();
//  }

};

    By adding functions to this object, you can add handling for specific routes. For example, if you uncommented the test function above (and restart the server), the server would respond to the route /test by outputting "Hello, World!". You can also specify what editor you want to use to edit the controller file by supplying an --editor option.

    $ nss edit --editor=vi

    Cleaning up

    When you are done with a server, you can get rid of it using nss destroy.

    $ nss destroy
